# 🐛 デバッグログ機能ガイド

## 概要

君斗りんくのWEBサイト健康診断には、**統合デバッグログ機能**が搭載されています。

ブラウザのコンソールログ（console.log, console.warn, console.error等）がすべて拡張機能内のデバッグログエリアにも表示されます。

---

## 📊 特徴

### 1. **統合表示**
- ブラウザのConsole（F12）
- 拡張機能のデバッグログエリア

**両方に同時に表示されます！**

### 2. **ログレベル別のアイコン**
- 📝 `console.log()` - 通常のログ
- ℹ️ `console.info()` - 情報
- ⚠️ `console.warn()` - 警告
- ❌ `console.error()` - エラー

### 3. **タイムスタンプ付き**
```
[8:30:15] 📝 SEO情報を生成しました
[8:30:16] ⚠️ サイトマップが見つかりません
[8:30:17] ❌ エラーが発生しました
```

### 4. **自動スクロール**
- 新しいログが追加されると自動的にスクロール
- 最新のログが常に見える

### 5. **コピー機能**
- 📋 コピーボタンで全ログをクリップボードにコピー
- サポートやバグ報告に便利

---

## 🚀 使い方

### 1. **デバッグログを表示**

1. ドメインを検索
2. **📊 SEO情報タブ**をクリック
3. 下部にスクロール
4. **🐛 デバッグログ**セクションが表示される

### 2. **ログをコピー**

1. デバッグログエリア内の **📋 コピー** ボタンをクリック
2. クリップボードにコピーされる
3. メールやチャットに貼り付け

### 3. **ログの見方**

```
[8:30:15] 📝 === SEOメタ情報取得開始 ===
[8:30:15] 📝 タイトル取得: むらきせい整骨院 | 整骨院の特徴
[8:30:16] ⚠️ サイトマップが見つかりません
[8:30:17] 📝 ✅ SEO基本情報をSEOタブに表示しました
```

- **[時刻]**: ログが記録された時刻
- **アイコン**: ログのレベル
- **メッセージ**: ログの内容

## 🔧 コンソールとの違い

### ブラウザのConsole（F12）
- **利点**: リアルタイム、詳細な情報、フィルタリング機能
- **欠点**: 拡張機能を閉じると消える

### デバッグログエリア
- **利点**: 拡張機能内で完結、簡単にコピー可能
- **欠点**: 基本的な情報のみ

**両方を併用するのがベスト！**

## 📋 ログの種類

### 📝 通常ログ (console.log)
```javascript
console.log('処理を開始しました');
```
→ `[8:30:15] 📝 処理を開始しました`

### ℹ️ 情報 (console.info)
```javascript
console.info('SEO情報を取得しました');
```
→ `[8:30:15] ℹ️ SEO情報を取得しました`

### ⚠️ 警告 (console.warn)
```javascript
console.warn('サイトマップが見つかりません');
```
→ `[8:30:15] ⚠️ サイトマップが見つかりません`

### ❌ エラー (console.error)
```javascript
console.error('エラーが発生しました:', error);
```
→ `[8:30:15] ❌ エラーが発生しました: Network error`
